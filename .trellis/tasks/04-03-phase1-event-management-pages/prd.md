# Phase 1 Event Management Pages

## Goal
Implement the phase-1 event management frontend pages using the approved information architecture and the lightweight emergency workspace approach.

## Requirements
- Replace the event list placeholder with a usable list page.
- Implement event edit page as a structured form page.
- Implement event detail page with phase-1 tabs or sections for event information, emergency progress, disposal records, and operation log.
- Represent lightweight response actions within the page rather than building full external integrations.
- Keep the work frontend-only with mock data and local interactions.

## Acceptance Criteria
- [ ] Event list page is implemented with search/filter/table behavior.
- [ ] Event edit page is implemented as a structured form page.
- [ ] Event detail page reflects the phase-1 lightweight emergency workspace concept.
- [ ] Loading / error / empty states are covered where relevant.
- [ ] `npm run build` passes in `frontend/`.

## Technical Notes
- Development type: frontend
- Keep collaboration, escalation, and response actions UI-only in phase 1.
- Do not integrate real external systems.
