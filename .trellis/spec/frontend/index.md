# Frontend Specifications

> Frontend-specific hard constraints for React 18 UI and client-side architecture.

---

## Reading Order

For frontend work, read documents in this order:

1. `../shared/index.md` for the project-wide baseline
2. This frontend index for frontend scope
3. The relevant frontend specification files listed below
4. `../guides/index.md` only when advisory design support would help with implementation decisions

## Usage

Use the frontend specifications for UI implementation decisions, component architecture, hook design, client state handling, type safety, and frontend quality standards.
Shared specifications remain the baseline for cross-cutting concerns such as code style, testing, configuration, API contracts, data modeling, deployment, workflow, and AI generation rules.

**Language**: All documentation should be written in **English**.
